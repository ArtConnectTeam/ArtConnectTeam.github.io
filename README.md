# ArtConnect pre-launch landing

Модульный pre-launch лендинг ArtConnect. Проект собран из HTML-компонентов через Jinja2, но итоговую страницу можно открыть как обычный статический HTML без бэкенда.

## Как открыть сайт

Самый простой вариант:

1. Откройте файл `index.html` в браузере.
2. Если меняли компоненты, сначала пересоберите файл командой:

```bash
python build.py
```

После сборки `build.py` обновит `index.html` и `index.built.html`.

## Зависимости

Для сборки нужен Python и библиотека `jinja2`.

Если `jinja2` не установлена:

```bash
python -m pip install jinja2
```

## Что за что отвечает

`index.html`  
Готовая статическая версия сайта для браузера и GitHub Pages. Этот файл пересоздается командой `python build.py`.

`index.template.html`  
Главный Jinja2-шаблон. В нем подключены шрифты, `css/styles.css`, `js/main.js` и стоят инклюды компонентов:

```jinja2
{% include 'components/hero.html' %}
```

`index.built.html`  
Дублирующая собранная версия сайта. Оставлена как удобный артефакт локальной проверки.

`build.py`  
Локальный сборщик. Берет `index.template.html`, подставляет все файлы из `components/` и сохраняет результат в `index.html` и `index.built.html`. Также меняет абсолютные пути `/css/...` и `/js/...` на относительные, чтобы статический файл работал при открытии с диска.

`css/styles.css`  
Все стили сайта: переменные `:root`, сетки, карточки, декоративные рамки, анимации, адаптивные media queries, стили hero-блока, секций, футера и интерактивных карточек.

`js/main.js`  
Клиентская логика:

- reveal-анимации при появлении блоков в viewport;
- прогресс-бар скролла;
- подсветка выбранной карточки заявки при переходе из CTA-кнопок;
- tilt-эффект для элементов с `data-tilt`;
- учет `prefers-reduced-motion`.

`assets/media/`  
Все изображения и декоративные медиа. CSS обращается к ним через `../assets/media/...`, HTML-компоненты через `assets/media/...`.

## Компоненты

`components/header.html`  
Верхняя декоративная рамка, sticky-шапка и навигация.

`components/hero.html`  
Главный экран: оффер, CTA-кнопки, декоративные элементы и интерактивный макет телефона.

`components/steps.html`  
Секции `Почему это нужно` и `Как это работает`. Здесь описан путь пользователя и механика культурного мэтчинга.

`components/audience.html`  
Секция аудиторий: зрители, мастера, организации, инвесторы/партнеры.

`components/masters.html`  
Отдельный блок для мастеров народных промыслов с баннером и списком преимуществ.

`components/feedback-forms.html`  
Партнерский блок и финальная секция заявок. В конце оставлены 4 карточки-ссылки:

- `Ранний доступ`
- `Я мастер`
- `Организация / событие`
- `Инвестор / партнер`

Сейчас ссылки ведут на заглушки `https://forms.gle/...`. Когда появятся реальные Google Forms, достаточно заменить `href` у этих четырех ссылок.

`components/footer.html`  
Декоративное поле внизу страницы и футер.

## Как менять контент

Для правки конкретной секции открывайте соответствующий файл в `components/`. После изменений выполните:

```bash
python build.py
```

И проверяйте результат в `index.html`.

## Важные заметки

- Не редактируйте `index.html` и `index.built.html` как основной источник: это результат сборки.
- Если добавляете новую картинку, кладите ее в `assets/media/`.
- Если добавляете новый компонент, подключите его в `index.template.html` через `{% include ... %}`.
- Для будущего минимального бэкенда можно использовать `index.template.html` как Jinja2-шаблон.
